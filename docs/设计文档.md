**版本**: 1.1.0
**日期**: 2026-02-28
**状态**: 草案

## 1. 项目概述

### 1.1 背景

OpenClaw 是一个自托管的 AI 编码代理网关，支持通过 WhatsApp、Telegram、Discord、iMessage 等即时通讯工具与 AI Agent 交互。其 Skill 系统允许开发者扩展 Agent 的能力边界，Plugin 系统允许注册自定义工具供 Agent 调用。

本项目目标是为 OpenClaw 构建一套完整的 Unity Editor 集成方案，使用户可以通过自然语言聊天消息，驱动 AI Agent 在 Unity Editor 中执行编辑器操作。

### 1.2 目标用户

- 使用 OpenClaw 的 Unity 开发者
- 希望通过 AI Agent 辅助 Unity 项目开发的团队
- 需要远程/移动端控制 Unity Editor 的开发者

### 1.3 核心价值

| 价值 | 说明 |
|------|------|
| 自然语言驱动 | 通过聊天消息描述意图，Agent 自动转译为 Unity 操作 |
| 无需打开编辑器界面 | 手机上也可以操作 Unity 场景 |
| 可组合操作 | Agent 可以链式调用多个工具完成复杂任务 |
| 可扩展 | 开发者可继续添加新的 Unity API 工具 |

## 2. 功能需求

### 2.1 场景操作

- **查询场景结构**: 获取当前打开场景的 GameObject 层级树
- **创建 GameObject**: 在场景中创建空对象或基础形状（Cube、Sphere、Plane 等）
- **删除 GameObject**: 按名称或路径删除场景中的对象
- **修改 Transform**: 设置对象的位置、旋转、缩放
- **父子关系管理**: 设置对象的父节点
- **查找对象**: 按名称、Tag、Layer 查找场景中的对象
- **组件操作**: 添加/移除/查询 GameObject 上的组件

### 2.2 资产管理

- **创建 C# 脚本**: 在指定路径生成新的 MonoBehaviour/ScriptableObject 脚本
- **创建材质**: 创建材质并设置 Shader
- **创建 Prefab**: 将 GameObject 保存为 Prefab
- **导入资产**: 触发 AssetDatabase.ImportAsset
- **刷新资产数据库**: 触发 AssetDatabase.Refresh
- **查询资产**: 按类型、路径、名称查找项目中的资产（含元数据：时长、尺寸、格式等）
- **资产详情**: 获取 Prefab 组件树及序列化字段值

### 2.3 编辑器控制

- **状态检查**: 检查 Unity Editor 运行状态、当前打开的场景
- **进入/退出 Play Mode**: 控制编辑器播放状态
- **保存场景**: 执行 EditorSceneManager.SaveScene
- **撤销操作**: 触发 Undo.PerformUndo
- **编译脚本**: 触发 AssetDatabase.Refresh 强制重编译

### 2.4 构建系统

- **触发构建**: 按指定平台执行 BuildPipeline.BuildPlayer
- **查询构建设置**: 获取当前构建目标平台、场景列表
- **修改构建设置**: 更新 Player Settings 中的配置项

### 2.5 代码执行（高级）

- **执行编辑器代码片段**: 接收 C# 表达式字符串，通过预编译代理类反射执行（危险操作，需明确授权）

### 2.6 代码自修正循环（核心能力）

Agent 写代码时不需要人工参与，系统自动形成闭环：

- **文件读取**: 读取项目内任意文件的当前内容（Agent 写完可立即读回确认）
- **文件写入**: 覆写已有文件或创建新文件，自动触发 Unity 重新导入
- **编译错误获取**: 获取最近一次编译的结构化错误列表（文件、行号、列号、信息）
- **编译状态查询**: 查询当前是否正在编译及结果
- **Console 日志获取**: 获取 Unity Console 的 Log/Warning/Error 输出
- **WebSocket 事件推送**: 编译完成/失败、Console 新增日志等事件实时推送到 Agent，无需轮询

### 2.7 项目理解与资源检索

Agent 开发新功能前需要理解已有项目，避免重复造轮子：

- **项目信息**: 项目名、Unity 版本、已安装 Package 列表
- **脚本 API 分析**: 列出所有用户脚本类及其公开方法/字段（编译成功后通过反射获取）
- **项目配置查询**: Tags、Layers、Input 轴、物理层设置
- **资源检索**: 按类型/名称/路径关键字搜索资产，返回含元数据的结果列表
- **引用反查**: 查询哪些 Prefab/Scene 依赖了指定资产（基于 AssetDatabase 依赖图）

## 3. 非功能需求

| 需求 | 要求 |
|------|------|
| 安全性 | HTTP Server 仅监听 localhost，不暴露外网 |
| 响应时间 | 普通操作 < 500ms，构建操作异步返回 |
| 线程安全 | 所有 Unity API 调用必须在主线程执行 |
| 稳定性 | Editor 重启后 Server 自动重新初始化 |
| 兼容性 | 支持 Unity 2021.3 LTS 及以上版本 |
| 错误反馈 | 所有错误返回结构化 JSON，包含错误码和描述 |

## 4. 用户交互示例

### 示例 1：查询场景

```
用户: 帮我看看当前场景里有哪些对象
Agent: 调用 unity_get_hierarchy
返回: 场景 "MainScene" 包含以下对象:
      ├── Main Camera
      ├── Directional Light
      └── Player
          ├── Body
          └── Weapon
```

### 示例 2：创建对象

```
用户: 在场景里添加一个名为 "Enemy_01" 的空对象，放在 (5, 0, 3) 位置
Agent: 调用 unity_create_gameobject + unity_set_transform
返回: ✓ 已创建 Enemy_01，位置已设置为 (5, 0, 3)
```

### 示例 3：创建脚本

```
用户: 帮我在 Scripts/Enemy 目录下创建一个 EnemyController 脚本
Agent: 调用 unity_create_script
返回: ✓ 已创建 Assets/Scripts/Enemy/EnemyController.cs
```

### 示例 4：构建

```
用户: 构建 Android 版本，输出到 Build/Android 目录
Agent: 调用 unity_build
返回: 构建已启动（后台运行），预计完成后通知
```

## 5. 约束与限制

- **不支持运行时（Play Mode）深度操作**: 主要面向编辑器状态，Play Mode 下的 Runtime API 不在当前范围
- **不执行任意 C# 代码**: exec 接口仅支持白名单操作集，防止代码注入风险
- **单 Editor 实例**: 当前设计不支持同时连接多个 Unity Editor 进程
- **本地运行限制**: Unity C# Plugin 必须在运行 OpenClaw Gateway 的同一台机器上，或通过 Tailscale 等方案实现内网穿透
- **脚本反射降级**: 编译失败时无法通过反射分析脚本 API，降级为返回文件路径列表
- **引用反查性能**: 大型项目（万级资产）的引用反查可能需要数秒，结果需缓存
- **非文本资产不做语义搜索**: 音频、贴图、动画等资产通过文件名关键字 + 目录约定 + 元数据过滤检索，不引入向量数据库；项目应遵循语义化目录结构（`Audio/BGM/`、`Audio/SFX/`、`UI/Prefabs/` 等）

## 6. 发布计划

| 阶段 | 内容 | 目标 |

## 7. 术语表

| 术语 | 说明 |
