### 常见问题

- Unity 版本 < 2022.3 需要 websocket-sharp（参见 unity-plugin 文档）
- 防火墙可能阻止 localhost 端口，需要临时关闭或添加例外
- `[InitializeOnLoad]` 在 Domain Reload 后会重新触发，HTTP/WS Server 会自动重启

### 1.3 入口文件

`src/index.ts` 的关键点：**必须在入口初始化 WebSocket**，compile 工具依赖它。

```typescript
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { UnityClient } from "./unity-client.js";
import { UnityWsClient } from "./unity-ws-client.js";
import { registerCompileTools } from "./tools/compile.js";
// ... 其他 import

const server = new McpServer({ name: "unity-editor", version: "1.0.0" });
const client = new UnityClient({
  port: Number(process.env.UNITY_PORT ?? 23456),
  timeout: 15000,
});
const ws = new UnityWsClient(Number(process.env.UNITY_WS_PORT ?? 23457));

// ⚠️ 必须在工具注册前连接，compile 工具依赖 WS 事件
ws.connect();

registerSceneTools(server, client);
registerGameObjectTools(server, client);
registerFileTools(server, client);
registerCompileTools(server, client, ws); // ws 传入 compile 工具
registerProjectTools(server, client);

const transport = new StdioServerTransport();
await server.connect(transport);
```

### 1.4 工具注册格式转换

从 `openclaw-plugin` 的注册格式迁移到 MCP SDK：

| 项目 | openclaw-plugin 格式 | MCP Server 格式 |
|------|---------------------|----------------|
| 注册方法 | `api.registerTool({name, description, parameters, execute})` | `server.registerTool(name, {description, inputSchema}, handler)` |
| 参数 schema | JSON Schema 对象 | `zod` 对象（SDK 自动转换） |
| 成功返回 | `return textResult("...")` | `return { content: [{ type: "text", text: "..." }] }` |
| 错误返回 | `return unityError(res)` | `throw new Error(res.error.message)` |
| 工具前缀 | 无（OpenClaw 直接用工具名） | `mcp__unity-editor__<工具名>`（Claude Code 调用时的前缀） |

**示例：`unity_get_hierarchy` 迁移**

```typescript
// 原 openclaw-plugin 格式
api.registerTool({
  name: "unity_get_hierarchy",
  description: "Get the full GameObject hierarchy of the current scene.",
  parameters: { type: "object", properties: {}, required: [] },
  execute: async (_id, _params) => {
    const res = await client.get("/scene/hierarchy");
    if (!res.ok) return unityError(res);
    return textResult(JSON.stringify(res.data, null, 2));
  }
});

// MCP Server 格式
server.registerTool("unity_get_hierarchy", {
  description: "Get the full GameObject hierarchy of the current scene.",
  inputSchema: {},  // 无参数时传空对象
}, async () => {
  const res = await client.get("/scene/hierarchy");
  if (!res.ok) throw new Error(res.error?.message ?? "Unknown error");
  return { content: [{ type: "text", text: JSON.stringify(res.data, null, 2) }] };
});
```

**示例：`unity_compile` 迁移**（含 WebSocket 等待）

```typescript
import { z } from "zod";

server.registerTool("unity_compile", {
  description: "Trigger Unity script compilation and wait for the result. Returns compile errors if any.",
  inputSchema: {
    timeoutSeconds: z.number().optional().describe("Max seconds to wait (default: 60)"),
  },
}, async ({ timeoutSeconds }) => {
  await client.post("/editor/compile");
  const timeoutMs = (timeoutSeconds ?? 60) * 1000;
  const result = await Promise.race([
    ws.waitForEvent("compile_complete", timeoutMs),
    ws.waitForEvent("compile_failed",   timeoutMs),
  ]) as { errors?: unknown[] };

  if (result?.errors?.length) {
    const errRes = await client.get<{ errors: unknown[] }>("/compile/errors");
    if (!errRes.ok) throw new Error(errRes.error?.message ?? "Failed to get errors");
    return { content: [{ type: "text", text: `Compilation FAILED:\n${formatCompileErrors(errRes.data.errors as any)}` }] };
  }
  return { content: [{ type: "text", text: "Compilation succeeded." }] };
});
```

### 1.5 MCP 配置文件

`mcp-server/claude-mcp-config.template.json`：

```json
{
  "mcpServers": {

      "env": {
        "UNITY_PORT": "23456",

      }
    }
  }

```

`install.sh` 在安装时将 `__MCP_SERVER_PATH__` 替换为实际路径后写入 `~/.claude/unity-mcp-config.json`。

### 1.6 install.sh

```bash
#!/bin/bash
# Unity MCP Skill 一键安装脚本

set -e

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
MCP_SERVER_DIR="$SCRIPT_DIR/../../mcp-server"

echo "=== Unity Editor MCP Skill 安装 ==="

# 检查依赖
command -v node  >/dev/null 2>&1 || { echo "❌ 需要 Node.js"; exit 1; }
command -v claude >/dev/null 2>&1 || { echo "❌ 需要 Claude Code CLI (claude)"; exit 1; }

# 构建 MCP Server
echo "📦 构建 MCP Server..."
cd "$MCP_SERVER_DIR"
npm install
npm run build

# 生成配置文件
CONFIG_PATH="$HOME/.claude/unity-mcp-config.json"
mkdir -p "$HOME/.claude"
sed "s|__MCP_SERVER_PATH__|$MCP_SERVER_DIR|g" \
  "$SCRIPT_DIR/claude-mcp-config.template.json" > "$CONFIG_PATH"

echo "✅ 配置已写入 $CONFIG_PATH"

# 验证
echo "🔍 验证安装..."
claude -p "echo installed" --mcp-config "$CONFIG_PATH" --output-format json --max-turns 1 > /dev/null 2>&1 \
  && echo "✅ Claude Code + MCP 配置验证通过" \
  || echo "⚠️  Claude Code 验证失败，请确认 Unity Editor 正在运行"

echo ""
echo "安装完成！在 OpenClaw 中说："
echo "  '在 Unity 场景里创建一个旋转的立方体'"
```

#### 步骤 2（任务 1.12）：编译自修正循环验证

这是**关键验收点**，验证 `unity_compile` 的 WebSocket 等待机制是否正常。

```bash
claude -p "请完成以下步骤：
1. 用 unity_write_file 在 Assets/Scripts/TestBrokenScript.cs 写入以下内容（故意缺少分号制造语法错误）：
   using UnityEngine;
   public class TestBrokenScript : MonoBehaviour {
       void Start() {
           Debug.Log(\"Hello\")   // 故意缺少分号
       }
   }
2. 用 unity_compile 触发编译并等待结果
3. 如果编译失败，用 unity_get_compile_errors 读取详细错误，修复文件（补上分号），再次编译
4. 编译成功后，用 unity_write_file 将 Assets/Scripts/TestBrokenScript.cs 内容清空删除（写入空字符串），或说明已完成
5. 输出整个过程的摘要" \
  --mcp-config ~/.claude/unity-mcp-config.json \
  --allowedTools "mcp__unity-editor__unity_write_file,mcp__unity-editor__unity_compile,mcp__unity-editor__unity_get_compile_errors,mcp__unity-editor__unity_read_file,mcp__unity-editor__unity_check_status" \
  --output-format json \
  --max-turns 15
```

**期望执行链路**：

```
unity_check_status()             → Unity 在运行
unity_write_file(有错误的代码)   → 写入文件，触发 AssetDatabase refresh
unity_compile()                  → 触发编译，阻塞等待 WS 事件
  ← compile_failed 事件到达      ← WebSocket 推送
unity_get_compile_errors()       → 读取具体错误（缺少分号）
unity_write_file(修正后的代码)   → 修复文件
unity_compile()                  → 再次编译
  ← compile_complete 事件到达   ← WebSocket 推送
输出：Compilation succeeded.
```

**验收标准**：
- `result` 描述了「写入错误代码 → 编译失败 → 读取错误 → 修复 → 编译成功」的完整流程
- `is_error` 为 `false`
- Unity Console 中能看到 `[OpenClaw] Server started on port 23456` 且无未捕获异常

#### 常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| `MCP server not found` | `dist/index.js` 不存在 | 重跑 `install.sh` |
| `Cannot connect to Unity Editor` | Unity 未运行或 Plugin 未导入 | 确认 Phase 0 通过 |
| `unity_compile` 60s 超时 | WebSocket 未连接（Domain Reload 未完成） | 等 Unity 编译完成后重试，或在 prompt 中加 `timeoutSeconds: 90` |
| `mcp__unity-editor__*` 通配符报错 | Claude Code 不支持该通配符 | 改为逐一列出工具名（见上方命令） |

## 阶段 3：SKILL.md 精调

### 3.1 SKILL.md 新增内容结构

在现有 `unity-editor/SKILL.md` 的"Code Self-Correction Loop"章节后追加：

```markdown
## Dispatching Implementation Tasks to Claude Code

For tasks requiring C# scripting, scene configuration, or multi-step Unity operations,
dispatch to Claude Code via exec, preserving context across sub-tasks with --resume.

### Task Decomposition Principle
Break large features into sub-tasks, each completable in ≤30 turns.
Example: "Player Controller" → ① Basic movement → ② Jump → ③ Coyote time → ④ Jump buffer

### First Call (new session)
[模板见架构文档]

### Subsequent Sub-tasks (resume context)
[模板见架构文档]

### Parsing Results
- `result` field: Claude Code's summary (⚠️ verify exact field name via real test)
- `session_id`: must be saved and passed to next sub-task's --resume

### When to Split Further
If a sub-task doesn't complete within 30 turns, split it into smaller pieces.
Do NOT increase --max-turns as a workaround.
```

### 3.2 项目状态快照机制

每次子任务完成后，OpenClaw 将结果 summary 追加到项目状态快照：

```
~/.claude/unity-project-state.md  （或保存在项目根目录）

# Unity Project State
## Implemented Features
- [x] PlayerController: basic movement (speed=5f) - Assets/Scripts/PlayerController.cs
- [x] PlayerController: jump (jumpForce=10f) - same file
- [ ] PlayerController: coyote time (pending)
```

后续任务通过 `--append-system-prompt` 注入快照内容，即使 session 过期也能恢复上下文。
