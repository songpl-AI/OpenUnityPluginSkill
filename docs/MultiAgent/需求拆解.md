# 需求拆解：OpenClaw + Claude Code + Unity MCP 多 Agent 系统

**版本**: 1.0.0
**日期**: 2026-03-02
**关联文档**: `../多Agent架构方案分析.md`

---

## 1. 背景与目标

用户通过 OpenClaw 对话，描述想要的游戏功能，系统自动完成 Unity 中的代码编写、编译修复、组件挂载，全程不需要人工干预。

**核心价值**：把"描述需求 → Unity 里跑起来"的链路从几小时缩短到几分钟，且不需要用户懂代码。

---

## 2. 用户故事

| ID | 角色 | 我希望... | 以便... |
|----|------|----------|--------|
| US-01 | 游戏开发者 | 用自然语言描述功能，AI 自动写好 C# 脚本并挂到 GameObject 上 | 不用手动写代码 |
| US-02 | 游戏开发者 | 编译出错时 AI 自动修复，不需要我介入 | 专注于设计而非调试 |
| US-03 | 游戏开发者 | 跨多个子任务时 AI 记得已有的代码结构 | 不用重复解释项目背景 |

## 3. 功能需求

### 3.1 Unity C# Plugin（已有，待验证）

| ID | 需求 | 验收标准 |
|----|------|---------|
| FR-01 | HTTP Server 在 Unity Editor 启动时自动运行 | `GET /api/v1/status` 返回 200 |
| FR-02 | WebSocket Server 推送编译事件 | 触发编译后收到 `compile_complete` 或 `compile_failed` 事件 |
| FR-03 | 编译错误详情可通过 API 获取 | `GET /api/v1/compile/errors` 返回带行号和消息的错误列表 |
| FR-04 | Domain Reload 后自动恢复（HTTP 重启，WS 重推） | 重新编译后仍能收到事件 |

### 3.2 MCP Server（新增）

| ID | 需求 | 验收标准 |
|----|------|---------|
| FR-05 | 封装所有 Unity 工具为 MCP 工具（共 19 个） | 含新增的 `unity_get_components`、`unity_add_component`、`unity_set_component_property` |
| FR-06 | 进程启动时建立 WebSocket 连接并全程保持 | `unity_compile` 工具能等到编译结果后返回 |
| FR-07 | WebSocket 断连后自动重连 | Domain Reload 期间的断连不影响后续工具调用 |
| FR-08 | 使用 stdio 传输协议 | 每次 `claude -p` 调用时按需启动，无需长驻 |

### 3.3 Claude Code 配置（新增）

| ID | 需求 | 验收标准 |
|----|------|---------|
| FR-09 | `unity-mcp-config.json` 配置 MCP Server | `claude -p "..." --mcp-config unity-mcp-config.json` 能找到并启动 MCP Server |
| FR-10 | `--output-format json` 输出结构稳定 | 每次都包含 `session_id` 和 `result` 字段（需实测确认字段名） |
| FR-11 | `--resume <session_id>` 恢复上下文 | 后续子任务能引用前一次任务写的文件和代码结构 |

### 3.4 OpenClaw Skill（更新）

| ID | 需求 | 验收标准 |
|----|------|---------|
| FR-12 | SKILL.md 描述顺序调用 + `--resume` 模式 | OpenClaw 能按模板拆分任务并传递 session_id |
| FR-13 | `install.sh` 一键完成 MCP Server 构建和配置 | 运行后 `claude -p "..." --mcp-config ~/.claude/unity-mcp-config.json` 可直接使用 |
| FR-14 | 任务粒度判断：子任务超限时拆分而非增大 max-turns | OpenClaw 能识别任务超限并重新拆解 |

## 4. 非功能需求

| ID | 类别 | 需求 | 指标 |
|----|------|------|------|
| NFR-01 | 可靠性 | WebSocket 断连自动重连 | 2s 内重连，不影响工具调用 |
| NFR-02 | 限制 | 单次子任务轮次上限 | `--max-turns 30`，超限则拆分任务 |
| NFR-03 | 安全性 | Claude Code 不能操作 Assets/ 外的文件 | `--allowedTools` 限定 + 系统提示声明禁区 |
| NFR-04 | 兼容性 | MCP Server 无需长驻进程 | 使用 stdio transport，按需启动 |
| NFR-05 | 可维护性 | MCP 工具逻辑复用 openclaw-plugin | 工具处理逻辑不重复实现，仅转换注册格式 |

## 5. 依赖与约束

### 强依赖（必须先完成）

```
Unity C# Plugin 集成测试通过（阶段 0）
    ↓
MCP Server 搭建完成（阶段 1）
    ↓
端到端链路验证（阶段 2）
    ↓
SKILL.md 精调（阶段 3）
```

### 外部依赖

| 依赖 | 说明 | 风险 |
|------|------|------|
| `@modelcontextprotocol/sdk` | MCP Server TypeScript SDK | 低，官方维护，稳定 |
| `claude` CLI | OpenClaw 通过 exec 调用 | 中，需确认 `--output-format json` 的字段名 |
| OpenClaw `exec` 工具 | 核心调用机制 | 低，已有其他 skill 验证可行 |
| OpenClaw `sessions_spawn` 工具 | 待调研，可能替代 exec 方案 | 待确认 |

## 6. 不在范围内

- 并行多实例 Claude Code（见架构文档第 4 节，留作未来扩展）
- Unity Runtime 运行时操控（仅 Editor）
- 非 Unity 项目的代码生成
- `tools/asset.ts`、`tools/editor.ts`、`tools/build.ts`（TypeScript 侧尚未实现的工具）
