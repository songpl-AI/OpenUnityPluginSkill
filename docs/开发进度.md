## 阶段五：集成测试

**状态**: 🔵 进行中（测试环境：Unity 6000.0.46f1 + macOS）

**前置条件**:
- [x] 将 `unity-plugin/Assets/OpenClawUnityPlugin/` 复制到目标 Unity 工程（TestOpenClaw）
- [x] 安装 `com.unity.nuget.newtonsoft-json` 3.2.1
- [x] OpenClaw Plugin：`npm install && npm run build` 完成，`--link` 安装到 OpenClaw
- [ ] Unity 2021.3：安装 websocket-sharp（Unity 6 不需要）

**已发现并修复的 Bug**（详见 `docs/FAQ.md` Q21-Q31）:
- `asmdef` 含无效 GUID 占位符 → `references` 改为 `[]`（Q21）
- `StatusHandler.cs` 缺 `using UnityEngine;` → 编译报错（Q22）
- `package.json` 缺 `"openclaw": {"extensions": [...]}` → 插件安装失败（Q23）
- `api.getConfig()` 不存在 → 改为 `api.pluginConfig ?? {}`（Q24）
- Mono HttpListener 拒绝 `localhost` → 统一使用 `127.0.0.1`（Q25）
- `Application.version` 返回项目版本 → 改为 `Application.unityVersion`（Q26）
- `StatusHandler` 中 `wsPort` 硬编码为 0 → 改为 `UnityEditorServer.HttpPort + 1`
- WS 前缀 `/ws/` 不匹配路径 `/ws` → 前缀改为 `/`（Q27）
- POST 无 body 时 Mono 返回 411 → 测试时需传 `-d '{}'`（Q28）
- `assets/Refresh` 不触发脚本编译 → 需调 `/editor/compile`（Q29）
- `parameters.required` 缺失 → 补充 `required: []` 防止框架 crash（Q30）
- `execute` 签名和返回格式不符框架要求 → 修正为 `(toolCallId, params)` + ToolResult（Q31）
