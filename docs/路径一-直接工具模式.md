# 路径一：OpenClaw 直接工具模式

**版本**: 1.0.0
**最后更新**: 2026-03-02

---

## 概述

OpenClaw 的 AI Agent（如 MiniMax、GLM、Kimi）在会话中**直接调用** Unity 工具，通过 HTTP 请求与 Unity Editor 通信。不涉及 Claude Code，也不涉及 MCP。

```
用户（飞书）
  → OpenClaw Agent（MiniMax / GLM / Kimi）
    → unity_check_status / unity_get_hierarchy / ...（直接工具调用）
      → openclaw-plugin/dist/index.js（TypeScript HTTP 客户端）
        → HTTP → http://127.0.0.1:23456/api/v1/...
          → Unity C# Plugin（HttpServer）
            → Unity Editor API
```

---

## 适用场景

| 场景 | 示例问题 |
|------|---------|
| 查询 Unity 运行状态 | "Unity 现在是否在运行？" |
| 读取场景结构 | "当前场景里有哪些 GameObject？" |
| 查看脚本信息 | "项目里有哪些脚本？PlayerController 有哪些公开字段？" |
| 查询编译状态 | "上次编译有没有错误？" |
| 查询项目信息 | "当前 Unity 版本是什么？安装了哪些 Package？" |
| 查找资源 | "项目里有没有叫 BGM 的音频文件？" |

> ⚠️ **路径一只适合只读操作。**
> 任何需要写文件、创建 GameObject、编译脚本、挂载组件的任务，必须使用路径二。

---

## 可直接调用的工具

| 工具 | 用途 |
|------|------|
| `unity_check_status` | 检查 Unity 是否运行、Plugin 是否激活 |
| `unity_get_scene_info` | 当前场景名称、路径、dirty 状态 |
| `unity_get_hierarchy` | 完整 GameObject 树 |
| `unity_get_scripts` | 所有脚本类及其公开 API |
| `unity_get_compile_errors` | 上次编译的错误列表 |
| `unity_get_console_logs` | Unity Console 输出 |
| `unity_get_project_info` | 项目名称、Unity 版本、Package 列表 |
| `unity_find_assets` | 按类型/名称搜索资源 |
| `unity_find_gameobjects` | 按名称/tag 查找 GameObject |
| `unity_get_components` | 查看某个 GameObject 的组件列表 |

## 安装方式

### 前置条件

- Node.js 18+
- OpenClaw 已安装并配置飞书频道
- Unity Editor 已打开

### 步骤 1：导入 Unity Plugin

将 `unity-plugin/Assets/OpenClawUnityPlugin/` 整个目录复制到你的 Unity 工程的 `Assets/` 下。

Unity 会自动编译并启动：
- HTTP Server：`http://127.0.0.1:23456`
- WebSocket Server：`ws://127.0.0.1:23457`

验证：
```bash
curl http://127.0.0.1:23456/api/v1/status
# 期望返回: {"status":"ok","unityVersion":"...","currentScene":"..."}
```

### 步骤 2：安装 OpenClaw 插件

```bash
# 进入插件目录
cd openclaw-plugin

# 安装依赖
npm install

# 构建
npm run build
```

### 步骤 3：注册到 OpenClaw

编辑 `~/.openclaw/openclaw.json`，在 `plugins` 字段中添加：

```json
{
  "plugins": {
    "load": {
      "paths": ["/path/to/openclaw-plugin"]
    },
    "entries": {
      "unity-editor": { "enabled": true }
    },
    "installs": {
      "unity-editor": {
        "source": "path",
        "sourcePath": "/path/to/openclaw-plugin",
        "installPath": "/path/to/openclaw-plugin",
        "version": "1.0.0",
        "installedAt": "2026-03-02T00:00:00.000Z"
      }
    }
  }
}
```

或使用 CLI：
```bash
openclaw plugin install /path/to/openclaw-plugin
```

### 步骤 4：重启 OpenClaw

```bash
openclaw restart
```

## 最佳实践

**1. 用路径一做"调查"，用路径二做"实施"**

工作流程示例：
```
用户: "给 Player 添加一个冲刺功能"
AI (路径一): unity_get_scripts → 了解现有 PlayerController 结构
AI (路径一): unity_get_hierarchy → 确认 Player 路径
AI (路径二): 委派给 Claude Code 实现冲刺逻辑
```

**2. 检查 Unity 状态后再委派**

每次任务开始，先调 `unity_check_status`，确认 Unity 在运行、Plugin 正常，再让 Claude Code 执行写操作。

**3. 配置合适的 LLM**

路径一的只读任务对模型要求不高，可以用速度快、成本低的模型（如 GLM-4.7 Flash）。

```json
"agents": {
  "defaults": {
    "model": { "primary": "zai/glm-4.7-flash" }
  }
}
```

## 常见问题

**Q: 调用工具返回 `SERVER_NOT_READY`**
A: Unity Editor 未运行，或 Plugin 未导入。打开 Unity 并等待编译完成。

**Q: 调用工具返回 `SCENE_NOT_LOADED`**
A: 没有打开的场景。在 Unity 中双击打开一个 Scene 文件。

**Q: `curl http://127.0.0.1:23456/api/v1/status` 返回 400**
A: 使用 `localhost` 而非 `127.0.0.1` 会因为 HttpListener Host 头校验失败。统一用 `127.0.0.1`。
